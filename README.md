# QuRating Methodology Reproduction with Qwen3.5-4B

This repository contains the reproduction of the data quality rating framework **QuRating** (ICML 2024 Spotlight), adapted to a **Qwen3.5-4B** (or Qwen3-4B fallback) backbone.

---

## 1. Quality Dimensions

We score text across four distinct quality dimensions:
1. **writing_style**: Fluency, coherence, vocabulary choice, and syntax.
2. **required_expertise**: Domain specificity and depth of knowledge required to comprehend the text.
3. **facts_and_trivia**: Density of factual information, events, data, and details.
4. **educational_value**: Pedagogical utility, clarity of explanations, and capacity to build skills or prompt critical thinking.

---

## 2. Bradley-Terry Preference Framework

Rather than training a generative SFT model, we model pairwise preferences. Given two text segments $A$ and $B$, the model assigns scalar ratings $s_A$ and $s_B$. The predicted probability that $B$ is superior to $A$ in dimension $d$ is:

$$P(B \succ A) = \sigma(s_B - s_A) = \frac{1}{1 + e^{-(s_B - s_A)}}$$

We train the model using a soft Binary Cross-Entropy (BCE) loss against the target preference judgments $y = P_{gt}(B \succ A) \in [0, 1]$ generated by LLM evaluators:

$$L = -[y \log \hat{y} + (1 - y) \log (1 - \hat{y})]$$

## 4. How to Run

### A. Smoke Test (Debug Pipeline Check)
Validate the environment, tensor flow, and optimization directions:
```bash
bash scripts/run_qurater_smoke.sh /path/to/Qwen3.5-4B
```

### B. Full Model Training (QLoRA)
Train the rating model with 4 independent classification heads andNF4 quantization:
```bash
bash scripts/run_qurater_train.sh /path/to/Qwen3.5-4B path/to/train_data.jsonl path/to/val_data.jsonl
```

### C. Evaluate Model
Calculate accuracy, BCE, AUC scores, and verify swap consistency ($p + p_{\text{swap}} \approx 1.0$):
```bash
bash scripts/run_qurater_eval.sh /path/to/Qwen3.5-4B /path/to/checkpoint/model.pt path/to/eval_data.jsonl
```

### D. Score Raw Corpus
Assign quality scores to raw text. Automatically chunks files exceeding 512 tokens using a sliding window and averages the scores weighted by window length:
```bash
python3 score_corpus.py \
    --model_path /path/to/Qwen3.5-4B \
    --checkpoint_path /path/to/checkpoint/model.pt \
    --input_file path/to/corpus.jsonl \
    --output_file path/to/scored_corpus.jsonl
```
